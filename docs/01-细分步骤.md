# 广告 SDK 接入细分步骤

本文档按 5 个步骤说明如何使用 `:advertise` SDK。每一步都包含需要调用的方法、推荐写法、代码样例和注意事项。

总体来说就是
1.从maven接入SDK
2.修改配置
3.开屏逻辑修改
4.加广告
5.打点 

本文档只写接入方法。`advertise` 库内部已经实现的广告策略、预加载池、Remote Config 路由、广告生命周期打点等能力，只在对应步骤中简要说明。完整功能说明可参考：[advertise库使用文档.md](../广告sdk功能文档/advertise库使用文档.md)。

## 第一步：引入 SDK，完成初始化

### 1.1 引入 SDK

如果 `:advertise` 是工程内模块，在 `settings.gradle.kts` 中确认已经包含：

```kotlin
// 引入 advertise 模块，宿主 app 才能依赖 SDK 对外 API。
include(":advertise")
```

在宿主 `app/build.gradle.kts` 中添加依赖：

```kotlin
dependencies {
    // 宿主通过 AdvertiseSdk 使用广告、打点、Remote Config、通知等能力。
    implementation(project(":advertise"))
}
```

如果 SDK 以 Maven 产物发布，依赖写法以实际发布坐标为准：

```kotlin
dependencies {
    // 版本号使用实际发布版本。
    implementation("com.pdffox:advertise:1.0.2") //接入具体版本
}
```

### 1.2 初始化 SDK

SDK 统一入口是 `AdvertiseSdk`，初始化方法是：

```kotlin
AdvertiseSdk.init(
    context = application,
    isTest = BuildConfig.DEBUG,
    sdkConfig = AdvertiseConfigFactory.create(application),
)
```

推荐在 `Application.onCreate()` 中尽早启动初始化：

```kotlin
class MyApplication : Application() {
    private val sdkInitScope = CoroutineScope(SupervisorJob() + Dispatchers.IO)

    override fun onCreate() {
        super.onCreate()

        sdkInitScope.launch {
            runCatching {
                AdvertiseSdk.init(
                    // 必须传 Application，SDK 内部会持有全局运行时上下文。
                    context = this@MyApplication,
                    // Debug 包传 true，SDK 会使用测试模式、日志和调试策略。
                    isTest = BuildConfig.DEBUG,
                    // 宿主集中创建完整 SDK 配置，避免页面层散落配置。
                    sdkConfig = AdvertiseConfigFactory.create(this@MyApplication),
                )
            }.onFailure { error ->
                // 初始化失败不能阻断 App 启动，但后续广告展示需要走降级路径。
                Log.e("SdkInit", "AdvertiseSdk init failed", error)
            }
        }
    }
}
```

### 1.3 初始化注意事项

1. `AdvertiseSdk.init()` 是 `suspend` 方法，必须在协程或其他挂起环境中调用。
2. SDK 初始化建议只在进程启动后执行一次。
3. 展示广告前要保证初始化流程已经启动，启动页可以等待一段时间后兜底继续。
4. 初始化传入的 `sdkConfig.adMob.appId` 必须和 Manifest 中的 AdMob App ID 一致。

## 第二步：进行相关配置

### 2.1 创建配置工厂

宿主应创建统一配置工厂，例如 `AdvertiseConfigFactory`：

```kotlin
object AdvertiseConfigFactory {
    fun create(context: Context): AdvertiseSdkConfig {
        return AdvertiseSdkConfigs.create(
            // context：宿主 Context，SDK 会用它读取包名并生成默认配置。
            context = context,
            // isDebug：当前是否 Debug 环境，会影响默认 topic、安全校验和部分调试逻辑。
            isDebug = BuildConfig.DEBUG,
        ) {
            // 隐私政策和用户协议，用于合规入口和 UMP 流程。
            legal(
                // privacyUrl：隐私政策地址，用于启动页、设置页或合规入口展示。
                privacyUrl = BuildConfig.ADV_PRIVACY_URL,
                // termsUrl：用户协议地址，用于启动页、设置页或合规入口展示。
                termsUrl = BuildConfig.ADV_TERMS_URL,
            )

            // FCM 默认 topic，空值时 SDK 会按 debug/release 使用兜底 topic。
            defaultTopic(
                // value：宿主希望订阅的默认 FCM topic。
                value = BuildConfig.ADV_DEFAULT_TOPIC,
            )

            // 广告调试覆盖模式，可用于测试策略打开、关闭或按策略执行。
            adDebugOverride(
                // mode：广告调试模式，常见值为 POLICY、FORCE_OFF、FORCE_ON。
                mode = BuildConfig.ADV_AD_DEBUG_OVERRIDE_MODE,
            )

            // 本地兜底资源，Remote Config 未返回时仍能使用本地策略。
            
            //todo 文件恢复不变，其他的需要使用对应的策略文件
            resources(
                // adPolicyRawResId：普通广告策略，控制开屏、插屏、Banner、激励视频等广告位展示和频控。
                adPolicyRawResId = R.raw.ad_policy,
                // adLoadConfigRawResId：广告预加载策略，控制预加载开关、触发时机、池大小和缓存有效期。
                adLoadConfigRawResId = R.raw.adload_config,
                // nativeAdPolicyRawResId：Native 广告展示策略，控制 Native 广告位展示和频控。
                nativeAdPolicyRawResId = R.raw.native_ad_policy,
                // nativeAdIdsRawResId：Native 高价、中价、兜底广告位队列。
                nativeAdIdsRawResId = R.raw.native_ad_ids,
                // cloudCidrsRawResId：Google Cloud CIDR 列表，用于环境和 IP 判断。
                cloudCidrsRawResId = com.pdffox.adv.R.raw.cloud,
                // googleCidrsRawResId：Google CIDR 列表，用于判断当前 IP 是否属于 Google IP。
                googleCidrsRawResId = com.pdffox.adv.R.raw.google,
                // pushConfigRawResId：旧版 Push 配置资源，当前已废弃，通常传 0。
                pushConfigRawResId = 0,
            )
            

            // Firebase Analytics 和 FCM 开关。
            firebase(
                // analyticsEnabled：是否启用 Firebase Analytics 事件上报。
                analyticsEnabled = true,
                // messagingEnabled：是否启用 Firebase Messaging/FCM。
                messagingEnabled = true,
                // subscribeDefaultTopic：初始化后是否订阅 defaultTopic。
                subscribeDefaultTopic = true,
            )

            // Remote Config 开关；如远端配置加密，需要传入对应密钥。
            remoteConfig(
                // enabled：是否启用 Firebase Remote Config 拉取和应用。
                enabled = true,
                // encryptionKeyBase64：Remote Config 聚合配置的解密密钥；明文配置可传空字符串。
                encryptionKeyBase64 = BuildConfig.ADV_REMOTE_CONFIG_ENCRYPTION_KEY,
                // encryptionKeyId：Remote Config 密钥 ID，用于选择正确解密密钥。
                encryptionKeyId = BuildConfig.ADV_REMOTE_CONFIG_ENCRYPTION_KEY_ID,
            )
            //todo encryptionKeyId 使用 ADconfig 生成
            // encryptionKeyId 使用 ADconfig 生成，配置remoteconfig使用，可以先关闭remoteconfig进行配置

            // AdMob 广告位配置，App ID 必须与 Manifest meta-data 一致。
            //todo需要变
            adMob(
                // enabled：是否启用 AdMob 广告网络。
                enabled = true,
                // appId：AdMob App ID，必须和 Manifest 中 com.google.android.gms.ads.APPLICATION_ID 一致。
                appId = BuildConfig.ADV_ADMOB_APP_ID,
                // bannerId：默认 Banner 广告位 ID。
                bannerId = BuildConfig.ADV_ADMOB_BANNER_ID,
                // interstitialId：默认插屏广告位 ID。
                interstitialId = BuildConfig.ADV_ADMOB_INTERSTITIAL_ID,
                // nativeId：默认 Native 广告位 ID。
                nativeId = BuildConfig.ADV_ADMOB_NATIVE_ID,
                // openId：App Open 开屏广告位 ID。
                openId = BuildConfig.ADV_ADMOB_OPEN_ID,
                // rewardedId：激励视频广告位 ID。
                rewardedId = BuildConfig.ADV_ADMOB_REWARDED_ID,
                // nativeIdsJson：Native 多价格广告位队列 JSON，线上建议由正式广告位组成。
                nativeIdsJson = BuildConfig.ADV_ADMOB_NATIVE_IDS_JSON,
                // debugNativeIdsJson：Debug 模式 Native 测试广告位队列；使用该常量时需要 import com.pdffox.adv.DEFAULT_DEBUG_NATIVE_IDS_JSON。
                debugNativeIdsJson = DEFAULT_DEBUG_NATIVE_IDS_JSON,
            )

            // ThinkingData 业务事件和用户属性上报。
            thinking(
                // enabled：是否启用 ThinkingData。
                enabled = true,
                // appKey：ThinkingData 项目 App Key。
                appKey = BuildConfig.ADV_THINKING_APP_KEY,
                // serverUrl：ThinkingData 数据上报服务地址。
                serverUrl = BuildConfig.ADV_THINKING_SERVER_URL,
            )

            // Singular 归因和广告收入上报。
            singular(
                // enabled：是否启用 Singular 归因和收入上报。
                enabled = true,
                // apiKey：Singular API Key。
                apiKey = BuildConfig.ADV_SINGULAR_API_KEY,
                // secret：Singular Secret，正式项目不要随意暴露。
                secret = BuildConfig.ADV_SINGULAR_SECRET,
            )

            // 安全校验，正式包建议打开签名和包名校验。
            safe(
                // enabled：是否启用安全校验。
                enabled = true,
                // expectedSignatures：允许的签名 SHA-256，支持逗号、分号或换行分隔。
                expectedSignatures = BuildConfig.ADV_SAFE_EXPECTED_SIGNATURES,
                // rejectDebuggableBuilds：正式校验时是否拒绝 debuggable 包。
                rejectDebuggableBuilds = true,
                // rejectDebuggerAttached：正式校验时是否拒绝已连接调试器的运行环境。
                rejectDebuggerAttached = true,
                // killProcessOnFailure：安全校验失败时是否直接杀进程。
                killProcessOnFailure = true,
                // expectedPackageName：期望包名；通常使用当前宿主 applicationId。
                expectedPackageName = BuildConfig.APPLICATION_ID,
                // enforceInDebugBuilds：Debug 包是否也执行强校验；本地调试通常传 false。
                enforceInDebugBuilds = false,
            )
            //todo需要变

            // Play Integrity，如当前包未接入云项目可先关闭。
            playIntegrity(
                // enabled：是否启用 Google Play Integrity 校验。
                enabled = false,
                // cloudProjectNumber：启用 Play Integrity API 的 Google Cloud Project Number。
                cloudProjectNumber = 0L,//todo需要变
                // runInDebugBuilds：Debug 包是否也请求 Play Integrity；本地调试通常关闭。
                runInDebugBuilds = false,
            )
        }
    }
}



```

### 2.2 配置 Manifest

宿主必须声明 AdMob App ID：

```xml
<application>
    <!-- 必须与 AdvertiseSdkConfig.adMob.appId 一致。 -->
    <meta-data
        android:name="com.google.android.gms.ads.APPLICATION_ID"
        android:value="${advAdmobAppId}" />
</application>
```

如果启用 Facebook 相关能力，再补充：

```xml
<!-- Facebook App ID，占位符来自 app/build.gradle.kts。 -->
<meta-data
    android:name="com.facebook.sdk.ApplicationId"
    android:value="${advFacebookAppId}" />

<!-- Facebook Client Token，占位符来自 app/build.gradle.kts。 -->
<meta-data
    android:name="com.facebook.sdk.ClientToken"
    android:value="${advFacebookClientToken}" />
```

### 2.3 配置 BuildConfig 和占位符

宿主 `app/build.gradle.kts` 中应集中声明广告配置：

```kotlin
android {
    defaultConfig {

        val advAdmobAppId = "ca-app-pub-xxxxxxxxxxxxxxxx~xxxxxxxxxx"

        buildConfigField("String", "ADV_ADMOB_APP_ID", "\"$advAdmobAppId\"")
        buildConfigField("String", "ADV_ADMOB_OPEN_ID", "\"ca-app-pub-xxx/open\"")
        buildConfigField("String", "ADV_ADMOB_INTERSTITIAL_ID", "\"ca-app-pub-xxx/inter\"")
        buildConfigField("String", "ADV_ADMOB_BANNER_ID", "\"ca-app-pub-xxx/banner\"")
        buildConfigField("String", "ADV_ADMOB_NATIVE_ID", "\"ca-app-pub-xxx/native\"")
        buildConfigField("String", "ADV_ADMOB_REWARDED_ID", "\"ca-app-pub-xxx/rewarded\"")

        // Manifest 中的 ${advAdmobAppId} 会使用这里的值。
        manifestPlaceholders["advAdmobAppId"] = advAdmobAppId
    }
}
```

### 2.4 配置资源

至少准备以下 raw 资源：

| 资源 | 作用 |
| --- | --- |
| `ad_policy.json` | 控制开屏、插屏、Banner、激励视频等广告位是否展示及频控。 |
| `adload_config.json` | 控制各触发时机是否预加载，以及缓存池大小、缓存有效期。 |
| `native_ad_policy.json` | 控制 Native 广告位展示策略。 |
| `native_ad_ids.json` | 由 `config/product.json` 的 `advertising.admob.nativeIds` 生成 Native 高价、中价、兜底广告位队列，不直接编辑。 |

## 第三步：开屏逻辑相关处理

### 3.1 推荐启动流程

```text
SplashActivity.onCreate
  -> 启动 SDK 初始化或等待 Application 中的初始化任务
  -> 初始化 UMP 同意状态
  -> 申请通知权限
  -> 根据策略预加载广告
  -> 展示 UMP 弹窗
  -> 展示首个开屏广告
  -> 开屏关闭、失败、无广告或超时后进入首页/引导页
```

### 3.2 初始化 UMP

```kotlin
AdvertiseSdk.initConsent(activity) { success ->
    // success 表示 UMP 同意状态请求流程结束，不代表一定展示了弹窗。
    Log.d("Splash", "UMP init finished: $success")
}
```

启动页展示 UMP：

```kotlin
AdvertiseSdk.showSplashConsent(activity) {
    // UMP 弹窗关闭、无需展示或异常兜底后，继续进入开屏广告流程。
    showOpenAdOrContinue()
}
```

### 3.3 预加载开屏广告（原生和插屏也可以进行预加载）

```kotlin
if (AdvertiseSdk.canPreloadOpen(AdvertiseSdk.LOAD_TIME_OPEN_APP)) {
    // 只有策略允许时才预加载，避免无条件重复拉取广告。
    AdvertiseSdk.preloadOpen(
        context = activity,
        loadTimeKey = AdvertiseSdk.LOAD_TIME_OPEN_APP,
    )
}
```

### 3.4 展示开屏广告

```kotlin
private fun showOpenAdOrContinue() {
    var continued = false

    fun continueOnce() {
        if (continued) return
        continued = true
        // 开屏关闭、失败、无广告、策略不命中后，都必须继续启动流程。
        goNextPage()
    }

    AdvertiseSdk.showOpenAd(
        activity = this,
        areaKey = AdAreaKeys.Open.OPEN_PAGE,
        onCloseListener = AdvertiseSdk.OpenAdCloseListener {
            // 这里不能留空，否则启动页可能无法进入后续页面。
            continueOnce()
        },
        onLoadedListener = AdvertiseSdk.OpenAdLoadedListener {
            // 广告已经加载完成，SDK 会继续展示。
            Log.d("Splash", "open ad loaded")
        },
        onPaidListener = AdvertiseSdk.OpenAdPaidListener { valueMicros ->
            // 收入回调，广告收入上报 SDK 内部已经处理；宿主一般不需要重复上报。
            Log.d("Splash", "open ad paid: $valueMicros")
        },
    )
}
```

建议外层加超时兜底：

```kotlin
lifecycleScope.launch {
    val completed = withTimeoutOrNull(6_500L) {
        suspendCancellableCoroutine<Unit> { continuation ->
            AdvertiseSdk.showOpenAd(
                activity = this@SplashActivity,
                areaKey = AdAreaKeys.Open.OPEN_PAGE,
                onCloseListener = AdvertiseSdk.OpenAdCloseListener {
                    if (continuation.isActive) continuation.resume(Unit)
                },
                onLoadedListener = AdvertiseSdk.OpenAdLoadedListener { },
                onPaidListener = AdvertiseSdk.OpenAdPaidListener { _ -> },
            )
        }
    }

    // 超时或广告流程结束后都进入后续页面。
    goNextPage()
}
```

### 3.5 前后台开屏控制

```kotlin
// 允许 SDK 监听前后台切换并展示 App Open 广告。
AdvertiseSdk.isAppOpenAdEnabled = true

// 从权限页、系统设置、文件选择器返回前设置，避免回到 App 立刻弹开屏。
AdvertiseSdk.suppressNextAppOpenAd = true
```

## 第四步：广告逻辑

### 4.1 定义广告位 key

宿主建议集中维护广告位 key：

```kotlin
object AdAreaKeys {
    object Open {
        // 首次冷启动开屏广告。
        const val OPEN_PAGE = "openPageAdv"
    }

    object Interstitial {
        // 进入功能页前展示的插屏。
        const val ENTER_FEATURE = "enterFeatureAdv"

        // 功能完成后展示的插屏。
        const val FEATURE_FINISH = "featureFinishAdv"
    }

    object Banner {
        // 首页底部 Banner。
        const val HOME_BOTTOM = "homeBottomBannerAdv"
    }

    object Native {
        // 首页 Native 广告。
        const val HOME = "homeNativeAdv"
    }

    object Rewarded {
        // 激励视频广告。
        const val REWARDED_VIDEO = "rewardedVideoAdv"
    }
}
```

这些 key 必须和 `ad_policy.json`、`native_ad_policy.json` 或 Remote Config 中的广告位配置一致。

### 4.2 插屏广告

插屏展示方法：

```kotlin
AdvertiseSdk.showInterstitialAd(
    activity = activity,
    areaKey = AdAreaKeys.Interstitial.ENTER_FEATURE,
) {
    // 插屏关闭、加载失败、策略不命中、无广告时都会继续执行这里。
    startActivity(Intent(activity, FeatureActivity::class.java))
}
```

建议封装为业务语义方法：

```kotlin
object AdvertisePageMediator {
    fun showEnterFeatureInterstitial(activity: Activity, onClosed: () -> Unit) {
        AdvertiseSdk.showInterstitialAd(
            activity = activity,
            areaKey = AdAreaKeys.Interstitial.ENTER_FEATURE,
        ) {
            // 所有未展示或关闭路径都通过 onClosed 继续原业务。
            onClosed()
        }
    }
}
```

插屏广告点梳理表建议保留在文档中：

| 场景 | 触发时机 | `areaKey` | 关闭后动作 |
| --- | --- | --- | --- |
| 进入功能页 | 点击首页功能入口 | `enterFeatureAdv` | 进入功能页 |
| 功能完成 | 点击完成或结果页跳转前 | `featureFinishAdv` | 进入结果页或推荐页 |
| 返回首页 | 功能页点击返回首页 | `returnHomePageAdv` | 返回首页 |

### 4.3 Banner 广告

View 系统：

```kotlin
fun bindBanner(context: Context, container: FrameLayout) {
    container.removeAllViews()

    val banner = AdvertiseSdk.getBannerAd(
        context = context,
        areaKey = AdAreaKeys.Banner.HOME_BOTTOM,
    )

    if (banner == null) {
        // 无广告、策略不命中或广告网络关闭时隐藏容器。
        container.visibility = View.GONE
        return
    }

    container.visibility = View.VISIBLE
    container.addView(
        banner,
        FrameLayout.LayoutParams(
            ViewGroup.LayoutParams.MATCH_PARENT,
            ViewGroup.LayoutParams.WRAP_CONTENT,
        ),
    )
}
```

销毁 Banner：

```kotlin
fun clearBanner(container: FrameLayout) {
    for (index in 0 until container.childCount) {
        // AdView 需要主动 destroy，避免离开页面后继续占用广告资源。
        AdvertiseSdk.destroyBannerAd(container.getChildAt(index) as? ViewGroup)
    }
    container.removeAllViews()
    container.visibility = View.GONE
}
```

Compose：

```kotlin
@Composable
fun HomeBanner() {
    val context = LocalContext.current
    val banner = remember {
        // remember 避免重组时重复创建 Banner View。
        AdvertiseSdk.getBannerAd(context, AdAreaKeys.Banner.HOME_BOTTOM)
    }

    banner?.let {
        BannerAd(adView = it, modifier = Modifier.fillMaxWidth())
    }
}
```

### 4.4 Native 广告

Compose 接入：

```kotlin
@Composable
fun HomeNativeAd() {
    val nativeAdState = rememberNativeAd(
        areaKey = AdAreaKeys.Native.HOME,
        onAdClicked = {
            // 这里是 AdMob 确认点击后的回调，可做业务侧统计。
            AdvertiseSdk.logEvent("native_ad_clicked", mapOf("areaKey" to AdAreaKeys.Native.HOME))
        },
    )

    nativeAdState.value?.let { nativeAd ->
        // 宿主需要用 NativeAdView 正确绑定素材，不能用普通 View 伪装广告。
        DisplayNativeAd(nativeAd)
    }
}
```

View 或非 Compose 接入：

```kotlin
val adGroup = AdvertiseSdk.getNativeAd(
    context = context,
    areaKey = AdAreaKeys.Native.HOME,
    onAdGroupLoaded = {
        // 广告池异步加载完成后刷新 UI，再次尝试取广告。
        refreshNativeContainer()
    },
    onAdClickListener = AdvertiseSdk.NativeAdClickListener {
        // 仅在 AdMob 确认点击后触发。
        AdvertiseSdk.logEvent("native_ad_clicked", mapOf("areaKey" to AdAreaKeys.Native.HOME))
    },
)

val nativeAd = adGroup?.hAd ?: adGroup?.mAd ?: adGroup?.lAd
if (nativeAd != null) {
    // 宿主将 NativeAd 绑定到 NativeAdView。
    showNativeAd(nativeAd)
}
```

注意：Native 广告必须使用 Google Mobile Ads 的 `NativeAdView` 注册素材 View。不要给 CTA 或素材 View 单独设置普通点击监听替代 SDK 点击回调，否则可能影响广告点击归因。

### 4.5 激励视频广告

```kotlin
AdvertiseSdk.showRewardedAd(
    activity = activity,
    areaKey = AdAreaKeys.Rewarded.REWARDED_VIDEO,
    onCloseListener = AdvertiseSdk.RewardedAdCloseListener {
        // 激励视频关闭或未展示后执行。不要在这里直接发奖励。
        Log.d("Rewarded", "rewarded ad closed")
    },
    onLoadedListener = AdvertiseSdk.RewardedAdLoadedListener {
        // 激励视频加载成功。
        Log.d("Rewarded", "rewarded ad loaded")
    },
    onPaidListener = AdvertiseSdk.RewardedAdPaidListener { valueMicros ->
        // 收入回调，SDK 内部已经处理广告收入上报。
        Log.d("Rewarded", "rewarded ad paid: $valueMicros")
    },
    onRewardListener = AdvertiseSdk.RewardedAdRewardListener { type, amount ->
        // 只有收到奖励回调后才发放业务奖励。
        grantReward(type, amount)
    },
)
```

### 4.6 广告预加载

预加载应该先判断策略：

```kotlin
if (AdvertiseSdk.canPreloadInterstitial(AdvertiseSdk.LOAD_TIME_ENTER_FEATURE)) {
    AdvertiseSdk.preloadInterstitial(
        context = context,
        loadTimeKey = AdvertiseSdk.LOAD_TIME_ENTER_FEATURE,
    )
}

if (AdvertiseSdk.canPreloadNative(AdvertiseSdk.LOAD_TIME_OPEN_APP)) {
    AdvertiseSdk.preloadNative(
        context = context,
        loadTimeKey = AdvertiseSdk.LOAD_TIME_OPEN_APP,
    ) {
        // Native 广告组加载完成后，可通知页面刷新广告容器。
        refreshNativeContainer()
    }
}

if (AdvertiseSdk.canPreloadRewarded(AdvertiseSdk.LOAD_TIME_ENTER_FEATURE)) {
    AdvertiseSdk.preloadRewarded(
        context = context,
        loadTimeKey = AdvertiseSdk.LOAD_TIME_ENTER_FEATURE,
    )
}
```

常用触发时机：

| 常量 | 含义 |
| --- | --- |
| `LOAD_TIME_OPEN_APP` | 打开 App。 |
| `LOAD_TIME_ENTER_FEATURE` | 进入功能页。 |
| `LOAD_TIME_PLAY_FINISH` | 广告播放完成或展示后补池。 |
| `LOAD_TIME_ENTER_BACKGROUND` | App 进入后台。 |
| `LOAD_TIME_RECEIVE_NOTIFICATION` | 收到通知。 |
| `LOAD_TIME_UNKNOWN` | 未指定来源。 |

### 5.5 广告生命周期打点说明

广告加载、加载完成、展示、点击、关闭、失败、超时、收入等事件由 `advertise` 库内部统一上报。宿主不要重复手动上报这些广告生命周期事件，否则会造成数据重复。

宿主只需要上报业务事件，例如入口点击、页面展示、功能完成、授权结果、通知点击等。
